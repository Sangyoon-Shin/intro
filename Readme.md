# I AM

임베디드 시스템 전공에서 금융권 백엔드 개발자로 새로운 방향을 찾아가는 과정을 담은 자기소개 웹사이트입니다.

단순한 프로필 나열이 아니라, 현재의 나와 SSAFY 지원 계기, 프로젝트 경험, 앞으로의 목표가 자연스럽게 이어지도록 구성했습니다.

---

## 기획

### 프로젝트 개요

- **프로젝트명:** I AM
- **주제:** 나 자신과 개발 경험 소개
- **형태:** MPA(Multi Page Application)
- **목적:** 나의 배경, 진로 변화 과정, 프로젝트 경험, 미래 목표를 웹사이트로 표현
- **개발 방식:** HTML, CSS, JavaScript를 활용한 바닐라 웹 개발
- **디자인 방향:** 차분한, 실용적인, 진정성 있는 개발자 포트폴리오

## 주요 페이지

상단 네비게이션은 다음 순서로 구성했습니다.

```text
HOME | PROFILE | JOURNEY | PROJECTS | FUTURE
```

### HOME

- 이름 및 핵심 캐치프레이즈
- 현재 상태와 목표 직무
- SSAFY, Backend, Java, Financial IT 키워드
- 다른 페이지로 이동하는 네비게이션

### PROFILE

기본 정보와 나의 성향을 소개합니다.

- 이름: 신상윤
- 나이: 26살
- MBTI: ISTP
- 전공: 임베디드 시스템
- 현재: SSAFY 교육생
- 목표: 금융권 백엔드 개발자

나를 표현하는 특징은 다음과 같습니다.

- 직접 실행하며 배우기
- 문제의 원인을 끝까지 확인하기
- 화려함보다 실용성을 중요하게 생각하기

### JOURNEY

임베디드 시스템 전공에서 금융권 백엔드 개발자를 목표로 하게 된 과정을 타임라인으로 표현했습니다.

- 임베디드 시스템 전공
- 서비스 개발에 대한 관심
- 금융 IT를 목표로 설정

### PROJECTS

지금까지 진행한 프로젝트 경험을 소개합니다.

#### 1. YOLO를 활용한 클라이밍 입문자용 레이저 안내 시스템

클라이밍 입문자가 다음 홀드의 위치를 쉽게 확인할 수 있도록 객체 탐지 결과와 레이저 안내 장치를 연결한 프로젝트입니다.

- YOLO를 활용한 클라이밍 홀드 객체 탐지
- MediaPipe를 활용한 사용자 신체 위치 인식
- Jetson Nano에서 객체 탐지 모델 실행
- Arduino와 레이저 장치를 활용한 위치 안내

**사용 기술**

- Python
- YOLO
- MediaPipe
- Jetson Nano
- Arduino

#### 2. 교내 단과대 전용 웹앱 프로젝트

교내 단과대 구성원을 위한 웹 서비스를 제작한 팀 프로젝트입니다.

**사용 기술**

- React

#### 3. 은행 거래내역서 기반 가계부 프로젝트

은행 거래내역서를 등록하고 거래 데이터를 관리·분석할 수 있도록 제작한 개인 프로젝트입니다.

- 은행 거래내역 데이터 등록 및 관리
- 거래내역서 기반 데이터 처리
- 월별 수입 및 지출 분석
- 카테고리별 소비 분석

**사용 기술**

- Java
- Spring Boot
- Spring Data JPA
- MySQL

### FUTURE

SSAFY에서 이루고 싶은 목표와 앞으로의 개발 방향을 소개합니다.

- Java 기본기 강화

최종적으로 복잡한 금융 비즈니스 로직과 데이터를 정확하고 안정적으로 처리할 수 있는 금융권 백엔드 개발자로 성장하는 것이 목표입니다.

## 폴더 구조

```text
I-AM/
├─ index.html
├─ html/
│  ├─ profile.html
│  ├─ journey.html
│  ├─ projects.html
│  └─ future.html
├─ css/
│  └─ style.css
├─ js/
│  └─ script.js
└─ README.md
```

### 경로 구성

루트의 `index.html`에서는 다음과 같이 CSS와 JavaScript를 연결합니다.

```html
<link rel="stylesheet" href="./css/style.css">
<script src="./js/script.js" defer></script>
```

`html` 폴더 내부 페이지에서는 다음 경로를 사용합니다.

```html
<link rel="stylesheet" href="../css/style.css">
<script src="../js/script.js" defer></script>
```

## 구현 과정

### 1. 콘텐츠 기획

나의 기본 정보, SSAFY 지원 계기, 프로젝트 경험, 미래 목표를 정리했습니다.

### 2. 디자인 콘셉트 설정

차분하고 실용적인 분위기를 중심으로 딥 네이비와 블루 계열의 색상을 선택했습니다.

### 3. 초기 SPA 제작

하나의 HTML 페이지 안에서 전체 콘텐츠 구조와 디자인을 먼저 구현했습니다.

### 4. MPA 구조로 확장

각 콘텐츠를 다음 페이지로 분리했습니다.

- HOME
- PROFILE
- JOURNEY
- PROJECTS
- FUTURE

### 5. 공통 스타일 및 동작 분리

HTML 내부에 작성된 CSS와 JavaScript를 각각 별도 파일로 분리했습니다.

- `css/style.css`
- `js/script.js`

### 6. Git을 활용한 버전 관리

기능과 페이지를 추가하는 과정마다 변경 내용을 커밋하여 관리했습니다.

## 주요 구현 포인트

### MPA 상대 경로 관리

루트의 `index.html`과 `html` 폴더 내부 페이지는 파일의 위치가 다르기 때문에 CSS, JavaScript, 다른 HTML 페이지의 상대 경로를 구분했습니다.

### 공통 네비게이션

각 페이지에 동일한 네비게이션을 적용하고 현재 페이지를 강조하여 사용자가 현재 위치를 쉽게 확인할 수 있도록 했습니다.

### 프로젝트 중심 기술 표현

기술 이름만 단독으로 나열하지 않고 각 기술을 실제로 사용한 프로젝트와 함께 제시했습니다.

### 정보 전달 중심 디자인

지나치게 화려한 효과보다는 방문자가 나의 배경과 목표를 쉽게 이해할 수 있도록 콘텐츠의 가독성을 우선했습니다.
